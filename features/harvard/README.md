# features/harvard

Harvard Sentences library — 72 phonetically-balanced lists used as the foundation of Lux's pronunciation practice corpus. This folder owns the "random next Harvard list" picker, the library modal (filter by phoneme, favorites, passage detail), and the state + favorites that back them.

## Key Files

- `index.js` — `wireHarvardPicker()`. Controls the picker button, a "random bag" shuffle (so every list is seen once per cycle), and `K_HARVARD_LAST` / `K_HARVARD_RANDOM_BAG` persistence. Lazy-loads the Library modal on first open.
- `modal.js` — Thin entrypoint (`export { createHarvardLibraryModal }`).
- `modal-controller.js` — The modal orchestrator: tabs (Harvard lists / Passages), phoneme filter, hover preview card, practice launch.
- `modal-dom.js` / `modal-render-list.js` / `modal-dom-helpers.js` — DOM construction + rendering of the list/detail rows.
- `modal-phoneme-metrics.js` — Harvard ⇄ passage phoneme metadata: `getHarvardMeta`, `getAllTopPhonemes`, `countForHarvard`, `totalForKey` — all reading from the phoneme meta generated by `scripts/build-harvard-phonemes.mjs`.
- `modal-favs.js` / `modal-data.js` / `modal-actions.js` / `modal-controller/` — favorites persistence (`K_HARVARD_FAVS`), data loaders, filter-UI actions, and the sub-controller with interaction handlers + constants.

## Conventions

- **One random bag per cycle.** `readRandomBag` in `index.js` guarantees no repeat until all 72 lists have been drawn.
- **Phoneme metadata is build-time, not runtime.** Run `node scripts/build-harvard-phonemes.mjs` to regenerate after a corpus change — don't hand-edit `src/data/harvard-phoneme-meta.js`.
- **Passages and Harvard lists share the modal UI.** `countForKey` / `totalForKey` branch on `isHarvardKey`. Treat both paths as first-class.
