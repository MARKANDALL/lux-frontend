# features/harvard

The Harvard Sentences picker and library modal. Lux uses the standardized Harvard Sentences corpus (72 phonetically-balanced lists × 10 sentences) as its pronunciation library. This module wires the dropdown picker, lazy-loads the Harvard data, and (on demand) the heavier library modal where users browse / filter by phoneme content.

## Key Files

- `index.js` — `wireHarvardPicker` (dropdown wiring), `loadHarvardList`, random-bag picking, and lazy `ensureHarvardPassages()` from `src/data/`.
- `modal.js` — single re-export of `createHarvardLibraryModal` from `modal-controller.js` (kept as a stable import path).
- `modal-controller.js` — orchestrates the modal: data load, render, filter UI, favourites, phoneme-metric panels.
- `modal-data.js` — `loadHarvardListRecords`, `loadPassageRecords` for the modal's row data.
- `modal-phoneme-metrics.js` — phoneme metadata helpers (`getHarvardMeta`, `metaForKey`, `getAllTopPhonemes`, count/total functions) used for the per-list phoneme summary.
- `modal-favs.js` — favourites persistence (Harvard-key list, stored via `app-core/lux-storage`).
- `modal-render-list.js`, `modal-dom.js`, `modal-dom-helpers.js`, `modal-actions.js`, `modal-phoneme-rows.js` — DOM construction and event wiring.

## Conventions

- Harvard data is large; never import `src/data/harvard.js` directly — call `ensureHarvardPassages()` so first paint stays lean.
- Phoneme metadata comes from `src/data/harvard-phoneme-meta.js`, generated by [`scripts/build-harvard-phonemes.mjs`](../../scripts/build-harvard-phonemes.mjs). Rebuild after any list change.
- Picker dropdown ids: `#harvardSelect`, `#harvardLoadBtn` (see `index.js`).
- The modal is mounted lazily on first open — keep its imports out of the hot path.
