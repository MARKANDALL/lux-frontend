# public

Static assets served verbatim by Vite. Anything in here ships to the browser without bundling and is referenced by absolute path (`/convo-img/...`, `/vendor/...`).

## Layout

- `convo-img/` — scenario thumbnails (WebP/JPG) for the AI Conversations picker. `convo-img/thumbs/` holds the down-scaled versions generated by [`scripts/make-thumbs.mjs`](../scripts/make-thumbs.mjs).
- `convo-vid/` — short looping MP4 backdrops for each scenario (airport, banking, coffee, doctor, …).
- `assets/characters/` — per-scenario role portraits (`<scenario>-<role>.jpg`) for the chat header.
- `vendor/` — third-party libraries dropped in as static files instead of bundled: `d3.v7.min.js`, `d3.layout.cloud.js`, `wavesurfer-7.8.11.min.js`. Loaded on demand by the wordcloud and self-playback features.
- `vid/ph-front/` — phoneme demo videos (front-of-mouth shots) used by the phoneme-hover tooltip.
- `lux-popover.js` — tiny standalone popover script loaded directly by `index.html` for the UID popover. Mirrors the `K_IDENTITY_UID` constant — keep the literal in sync if it ever changes.

## Conventions

- Files here are NOT bundled. New images should be added to the matching folder and referenced by absolute path.
- Large new media should be downscaled or compressed before commit; this folder is the main contributor to repo weight.
- Vendor libs that have a working npm version belong in `package.json` instead — `vendor/` is reserved for libraries that misbehave under bundling.
