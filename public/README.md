# public

Vite's static-asset folder. Everything here is served verbatim at the site root. Nothing under `public/` is processed by the bundler, so references must use root-absolute paths (e.g. `/convo-img/airport.webp`, not an `import`).

## Key Files / Folders

- `convo-img/` — Scenario cover art for the AI Conversation picker (`airport.webp`, `banking.webp`, `calling.webp`, etc.). Thumbnails under `convo-img/thumbs/` are generated by `scripts/make-thumbs.mjs`.
- `convo-vid/` — Scenario background video loops for the picker stage.
- `vid/` — Page-level/hero video assets.
- `assets/characters/` — Character portraits for conversation NPCs.
- `vendor/` — Pinned third-party libs loaded as plain `<script>` or fetched at runtime: `d3.v7.min.js`, `d3.layout.cloud.js`, `wavesurfer-7.8.11.min.js`. Kept here so bundler quirks cannot move or tree-shake them.
- `lux-popover.js` — Standalone popover helper served directly from the site root.

## Conventions

- Reference assets by absolute URL (`/convo-img/foo.webp`), never via ES import.
- Regenerate thumbnails with `node scripts/make-thumbs.mjs` after adding scenario imagery.
- Do not hand-edit vendored libraries in `public/vendor/` — replace the whole file when upgrading.
