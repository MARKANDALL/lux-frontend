# public

Static assets served as-is by Vite (no bundling, no hashing). Contents are referenced from HTML and CSS via absolute paths (`/convo-img/...`, `/assets/...`).

## Contents

- `convo-img/` — scenario images for AI Conversations (`*.webp` / `*.jpg`); `thumbs/` holds the derived thumbnails generated by `scripts/make-thumbs.mjs`.
- `convo-vid/` — scenario video loops.
- `vid/` — miscellaneous instructional video clips.
- `assets/characters/` — scenario character art.
- `vendor/` — vendored third-party assets.
- `lux-popover.js` — a legacy popover script loaded via `<script>` tag; kept in `public/` because it predates the module pipeline.

## Conventions

- Do not bundle from `public/` — files here ship unchanged and appear at site root.
- Thumbnails under `convo-img/thumbs/` are derived artefacts — regenerate via `npm run thumbs`, do not hand-edit.
- Large binary additions should be checked against repo-size limits before committing.
