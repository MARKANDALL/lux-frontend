# public

Static assets served as-is. Anything in this folder is copied into the build output at the same path, so `public/lux-popover.js` is reachable as `/lux-popover.js` in the deployed app. Reserved for vendor scripts, media, and the UID popover — **not** app code.

## Key Files & Subfolders

- `lux-popover.js` — Tiny vanilla script loaded by the UID popover surface. Reads `LUX_USER_ID` from `window` or `localStorage` and offers a copy-to-clipboard affordance. (Key mirrors `K_IDENTITY_UID` from `app-core/lux-storage.js`.)
- `vendor/` — Pinned third-party scripts served from same-origin: `d3.v7.min.js`, `d3.layout.cloud.js` (wordcloud), `wavesurfer-7.8.11.min.js` (self-playback waveform).
- `convo-img/` — Scenario artwork (27 webp/jpg files: airport, banking, coffee, interview, etc.) plus a `thumbs/` subfolder generated by `scripts/make-thumbs.mjs`.
- `convo-vid/` — Matching scenario background videos (mp4).
- `assets/characters/` — Character portraits used in the Convo characters drawer.
- `vid/` — Other video assets.

## Conventions

- **No app code here.** Anything that should run through Vite's bundle belongs in `src/`, `features/`, or `ui/`.
- **Vendor scripts are pinned by version in the filename.** When upgrading, drop the new file alongside and update the `<script>` tag — do not hot-swap an unversioned URL.
- **`convo-img/thumbs/` is generated.** Run `node scripts/make-thumbs.mjs` to regenerate after adding or updating scenario artwork.
