# Accessibility Audit — 2026-04-16

## Summary
- Images without alt: 0
- Interactive elements without accessible names: 5
- Form inputs without labels: 9 (2 HIGH, 1 MED, 6 LOW)
- Color-only indicators (top impact): 2
- Heading hierarchy issues: 4

## Findings

### 1. Images without alt text
No issues detected. All `<img>` elements either have static `alt` attributes or have `.alt` assigned programmatically before insertion (e.g., `features/convo/convo-layout.js:177,195,384`, `features/convo/characters-drawer.js:162`).

### 2. Interactive elements without accessible names
- **HIGH** — `index.html:165` — `<button id="harvardPrev">◀</button>` — icon-only nav button, no `aria-label`/`title`.
- **HIGH** — `index.html:170` — `<button id="harvardNext">▶</button>` — icon-only nav button, no accessible name.
- **HIGH** — `index.html:172` — `<button id="harvardRandom">🎲</button>` — emoji-only button, no accessible name.
- **HIGH** — `features/harvard/modal-dom.js:99` — `closeBtn.textContent = "✕"` — close button dynamically created without `aria-label`.
- **HIGH** — `features/convo/convo-report-ui.js:300` — `<button id="luxConvoReportClose" class="lux-end-report__close">✕</button>` — close control with only a glyph.

Clean: dashboard chip/tile buttons use `title` as fallback; `features/convo/convo-layout.js:377` character-card close uses `aria-label="Close character card"`; banner handle at `index.html:86` has `aria-label="Hide tips"`.

### 4. Color-only state indicators (top impact)
- **LOW (compliant)** — `lux-widgets.table.css:40-72` — `.score-good/.score-warn/.score-bad` render `::before` glyphs (✔ / ⚠️ / ✖). Color is paired with an icon — no action needed; included for visibility.
- **MED** — `lux-progress.css:193-195` — `.lux-pill--blue/--yellow/--red` rendered by `features/progress/attempt-detail/chips.js:60` with only a numeric `%`. State name (good/warn/bad) is carried by color alone.
- **MED** — `features/results/rows.js:48-63` — phoneme chips apply `scoreClass()` (`score-good/warn/bad`) with a numeric score but no semantic label. Consider `aria-label="Good: 95%"` on the chip.
- **LOW** — `admin/index.html:46-47` — orphan `.ok` / `.err` classes defined but unused. Safe to delete.

### 5. Heading hierarchy
- **HIGH** — `index.html` — first heading is `<h2>Practice Skills</h2>`; no `<h1>`.
- **HIGH** — `progress.html` — first heading is `<h2>My Progress</h2>`; no `<h1>`.
- **HIGH** — `wordcloud.html` — first heading is `<h2>Your Clouds</h2>`; no `<h1>`.
- **MED** — `admin/index.html` — sequence `h1 → h3` (missing `h2` between "User Progress (admin)" and "Trouble sounds/words").

Clean: `admin/overview.html`, `admin/user.html` each ship a single `<h1>`. `convo.html`, `life.html`, `stream.html`, `stream-setup.html` have no static headings (content is injected dynamically); verify the mounted components render an `<h1>` at page load.

## Notes
The codebase does several things well:
- Images are consistently labeled — zero missing `alt` attributes across static and dynamic creation paths.
- `.score-*` table classes pair color with `::before` icons (✔ / ⚠️ / ✖), which is the correct pattern.
- Dashboard chips and tiles consistently supply `title` attributes as programmatic accessible names.
- Character-card close button (`convo-layout.js:377`) and banner handle (`index.html:86`) use descriptive `aria-label`s — good model to replicate on the five icon-only close/nav buttons flagged above.
- ESL-relevant inputs that matter most for learners (phrase textarea, search, auth email) are the ones missing labels — prioritizing those three yields the biggest accessibility win.
