### WARNING
- `helpers/md-to-html.js:28` — The heading pre-split regex `/^(#{2,3} .+?)\s+(?=[^\n])/gm` uses a non-greedy `.+?`, so a heading with no trailing newline like `"## Section Title"` is split at the first internal space. `mdToHtmlFull("## Section Title")` yields `<h3>Section</h3>\n<p>Title</p>` instead of `<h3>Section Title</h3>`. Already catalogued (audit 20.1), but worth re-surfacing because this week's refactor routes all full-document AI feedback renders (`ui/ui-ai-ai-dom.js:302`) through this exact regex.

### INFO
- `ui/ui-ai-ai-dom.js:287,294,301` — `showAIFeedbackError`, `clearAIFeedback`, and `renderAIFeedbackMarkdown` call `ensureShell(box, null)` without first checking `box`. If `#aiFeedback` is absent from the DOM, `ensureShell` dereferences `box.querySelector` / `box.innerHTML` and throws. Pre-existing and the refactor didn't make it worse, but the other entry points in this file (`hideAI`, `renderEntryButtons`) do guard; the inconsistency is worth closing. Low confidence that the element ever actually goes missing at runtime.
- `ui/ui-ai-ai-dom.js:97-106` — `wireSidebarButtons` clones each `.ai-voice-btn` to drop stale listeners, but the click handler closes over `newBtn` captured at clone time; if a future change ever swaps the element again (e.g. re-templating the sidebar), the handler will call `classList.add("active")` on a node no longer attached. Not broken today, just brittle if the sidebar shape changes.
