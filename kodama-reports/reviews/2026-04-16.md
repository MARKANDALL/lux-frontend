### CRITICAL

#### 1. `ui/auth-dom.js:140` — `email.split()` can throw when email is undefined

Some Supabase / OAuth providers (e.g. phone, anonymous, certain SSO flows) deliver a session without `session.user.email`. `email.split("@")` then throws `TypeError: Cannot read properties of undefined (reading 'split')`, which aborts the handler *before* `setUID(realUid)` runs. Net effect: the user appears signed in to Supabase but the app never adopts the real UID, and migration never fires.

**Before:**
```js
if (session?.user) {
  const email = session.user.email;
  const name = email.split("@")[0];
  ...
  btn.onclick = () => {
    if(confirm(`Signed in as ${email}.\n\nDo you want to log out?`)) {
```

**After:**
```js
if (session?.user) {
  const email = session.user.email || "";
  const name = email ? email.split("@")[0] : "account";
  ...
  btn.onclick = () => {
    const who = email || "this device";
    if (confirm(`Signed in as ${who}.\n\nDo you want to log out?`)) {
```

Confidence: high.

#### 2. `ui/auth-dom.js:133` — `onAuthStateChange` listener is never unsubscribed

`supabase.auth.onAuthStateChange()` returns a subscription; the code discards it. If `initAuthUI()` is ever called more than once (HMR, late mount, route-re-entry), each call attaches another listener. On the next `SIGNED_IN`/`TOKEN_REFRESHED` every attached listener will fire, which means `migrateHistory()` can be invoked N times concurrently with the same `(guestUid, realUid)` pair — duplicate POSTs to `/_api/migrate`.

**Before:**
```js
function handleAuthStateChange() {
  supabase.auth.onAuthStateChange(async (event, session) => {
    ...
  });
}
```

**After:**
```js
let _authSub = null;
function handleAuthStateChange() {
  if (_authSub) return;                       // guard against double-init
  const { data } = supabase.auth.onAuthStateChange(async (event, session) => {
    ...
  });
  _authSub = data?.subscription ?? null;
}
```

Confidence: high that duplicate attachment is possible; medium on whether it fires today (depends on real call sites of `initAuthUI`).

### WARNING

#### 3. `ui/auth-dom.js:156-165` — Migration may re-run on every auth event

`onAuthStateChange` fires for `SIGNED_IN`, `TOKEN_REFRESHED`, `USER_UPDATED`, `INITIAL_SESSION`, etc. The guard is `guestUid && guestUid !== realUid`. After migration, the code calls `setUID(realUid)` — but unless `setUID` also overwrites `K_IDENTITY_UID` in storage, the old guest UID remains and `migrateHistory()` re-runs on the next token refresh. I could not confirm `setUID`'s side effects from this file alone — please verify in `_api/index.js`.

Suggested mitigation: after a successful migration, explicitly `setString(K_IDENTITY_UID, realUid)` (or clear it), and early-return on events other than `SIGNED_IN`.

Confidence: medium — depends on `setUID` behavior.

#### 4. `ui/auth-dom.js:102-105` — No try/catch around `signInWithOtp`

If the call throws (network error, CORS, Supabase outage) rather than returning `{ error }`, the button is left in the "Sending..." / disabled state forever. Wrap in `try/catch` and restore button state in a `finally`.

Confidence: high.

#### 5. `features/results/summary.js:141-163` — Words interpolated into `innerHTML` unescaped

`err.word` originates upstream from `NBest[0].Words[].Word` (Azure Pronunciation Assessment). Azure normally returns plain tokens, but there's no guarantee the string is HTML-safe, and `$out.innerHTML = html` at line 171 commits it to the DOM. If a word ever contains `<`, `&`, or a quote, rendering breaks silently; a malicious mock response could inject script. Recommend wrapping `err.word` with `escapeHtml()` (already imported elsewhere in the codebase).

Confidence: medium. Real-world Azure payloads are almost certainly safe, but the invariant isn't documented.

#### 6. `features/results/summary.js:119` — `youtubeId` interpolated into iframe `src`

`passages[passageKey].youtubeId` comes from local data and is likely trusted, but the value is placed directly into an iframe URL without validation. A stray space or quote in `youtubeId` would break out of the attribute. Validate with a regex like `/^[A-Za-z0-9_-]{6,20}$/` before interpolating, or render the iframe via `createElement` + `setAttribute`.

Confidence: medium.

#### 7. `features/harvard/modal-actions.js:84-92, 122-129, 149-153, 169-173, 239-252` — Unchecked DOM deref after partial guards

Several functions guard some params (`if (!listEl) return`) but then dereference sibling params (`selTitle.textContent`, `practiceBtn.textContent`, `hoverTitle.textContent`, `hoverCard.style`) without null checks. If a caller passes a partial options object, you get a `TypeError` instead of a silent no-op. Either make the contract explicit (throw early with a useful message) or add `if (!selTitle || !practiceBtn) return;`.

Confidence: high for the missed checks; low on whether any current caller omits these params.

#### 8. `features/harvard/modal-actions.js:96, 133` — `globalThis.warnSwallow` may not exist

The `catch` blocks call `globalThis.warnSwallow(...)`. If that helper isn't installed on `globalThis` at the moment the handler runs (load order, test env), the catch block *itself* throws and the original `setString` error is replaced by `TypeError: globalThis.warnSwallow is not a function`. Safer:

```js
} catch (err) { globalThis.warnSwallow?.("features/harvard/modal-actions.js", err, "important"); }
```

Confidence: high.
