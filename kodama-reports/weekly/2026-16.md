**Top 3 refactor candidates (per Refactor Constitution v2, target ~200±100 LoC, ≤6 modules per area):**

1. **`lux-progress.css` (964)** — no data-file excuse; global CSS monolith. Natural peels: layout shell, cards/tiles, wordcloud-dock, responsive queries. Candidate for 4–5 smaller files under a `lux-progress.*.css` convention already in use for `lux-results.*` / `lux-widgets.*`.
2. **`features/features/tts.css` (948)** — TTS player styling. Sibling files (`tts-peekaboo.css`, `self-playback.css`) already sit next to it, so splitting into `tts.player.css`, `tts.controls.css`, `tts.karaoke.css` fits the existing pattern.
3. **`features/convo/scenarios.js` (657)** — even as data, it is close to a 10× overshoot of target and mixes 25 scenarios plus metadata. Candidate to shard one-file-per-scenario under `features/convo/scenarios/` with an `index.js` barrel, cutting per-file LoC to ~25–40 and keeping the directory under the ≤6-modules-per-area rule only if scenarios are grouped (by CEFR level or theme).
