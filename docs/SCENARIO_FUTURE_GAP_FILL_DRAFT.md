FUTURE SYSTEM IDEAS (catalog)
Emotion Preset Visualization System
Concept: When a user selects a scenario, they see a visual indicator showing how much emotional content is already built into that scenario. This could be:

Color intensity (light = neutral, dark = high emotion)
Size of emotion tags
A small meter or gauge next to each scenario card
Color-coded dots showing which emotions are naturally present

Why it matters: If a user selects "Happy/Jokey" tone for a funeral scenario, the system could show that the scenario already has high "serious/somber" emotion built in — creating a visual warning that these may conflict. This doesn't prevent the choice, but makes the user aware.
Implementation notes: Each scenario would need internal emotion metadata (not shown as text, but used for the visualization). Something like:
emotionPreset: { serious: 0.8, tense: 0.3, warm: 0.1 }
This would drive the visual indicator on the scenario picker card.
Status: Idea logged. Not scheduled. Requires design mockup, emotion metadata schema, and frontend implementation.

DISTRIBUTION PRINCIPLE
When the full set of 100+ scenarios is built, they must be distributed in the picker UI so that:

No two similar scenarios are adjacent. Parking and Police should not be back-to-back. Doctor and Pharmacy should be separated. All service/transaction scenarios should be spread across the deck.
Variety is maximized at every scroll position. A user scrolling through should see a mix of domains, channels, and tones within any 5-6 consecutive cards.
Ordering algorithm (proposed): Sort by domain, then interleave — take one from each domain in round-robin order. This automatically spreads similar scenarios apart.
Category tags (internal): Each scenario gets internal tags for domain, channel, tone, and difficulty level. These enable smart sorting, filtering, and recommendation ("You've done 4 service scenarios — try a social one next").
